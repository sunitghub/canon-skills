# Dataflow — Agent Instructions

## Code Style

- Use 2-space indentation.
- Prefer `async/await` over raw Promises.
- Always use `const` over `let` unless reassignment is needed.
- Write clean, readable code.
- Name things clearly.

## Testing

- Write tests for all new functionality.
- Tests must pass before opening a PR.
- Use Jest for unit tests.

## Git

- Use imperative mood in commit messages.
- Keep commits small and focused.
- Never force-push main.

## General

- Follow best practices.
- Think carefully before making changes.
- Make sure your work is complete before stopping.
