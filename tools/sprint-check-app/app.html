What's inside
Kanban board—your .tickets/ folder as a live board.
Four columns: Open, In Progress, Done, Discarded. Drag cards to update status instantly. Auto-refreshes every 8 seconds.
drag-to-move auto-refresh light & dark
Ticket completeness—readiness baked into every card.
Ready, incomplete, and needs-doc states show whether Acceptance has criteria/tests and Plan has real Approach notes. Hover the indicator for a checklist popover so doc problems surface before close.
acceptance gate test plan check inline warnings
Sprint docs—Acceptance and Plan attached to tickets, edited inline.
Open any ticket → click a doc tab to read or edit. Templates guide the format. The editor toolbar inserts checkboxes, bullets, headings, and code at the cursor.
acceptance.md plan.md in-browser editor
Commit intelligence—click any commit to see what it touched.
Direct ticket ID matches show first. When no ID is present, semantic keyword matching surfaces the most likely related ticket.

Create from the board—new tickets without leaving the browser.
The title auto-detects type (feature, task, bug, chore, epic). Set priority, add a description, hit Create — the ticket appears in .tickets/ immediately.
